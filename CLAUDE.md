# Ronin Advisory Group Website

## Role and Scope

Build the Ronin Advisory Group website. Greenfield, hand-written HTML/CSS, published to GitHub Pages. Dual-harness: primitives must be usable from Claude Code and Pi Agent. Project-specific slash commands, agents, and skills are prefixed `rag-web-*`. Priming is read-only; closing is adaptive dispatch; commits are operator-gated.

---

## Rules

- All project-specific slash commands, agents, and skills are prefixed `rag-web-*`.
- Hand-written HTML and CSS only. No static-site generator, framework, or build tooling without explicit approval.
- Publishing target is GitHub Pages. No features that require a server runtime.
- Both harnesses are first-class. When a Claude Code primitive ships, open a follow-up for the Pi equivalent — silent single-harness lock-in is a rule violation.
- `/rag-web-close` never auto-commits. It presents options and waits for the operator.
- `.the-grid/sessions/beacons/` and `.the-grid/sessions/hook_logs/` are ephemeral and git-ignored. `.the-grid/sessions/context_bundles/` and `.the-grid/sessions/summaries/` are tracked history.
- The machine environment profile lives at `.the-grid/config/<hostname>-SYSTEM-ENV.md` and is regenerated by `/prime-env`. Session primes read it; they do not regenerate it.

## Antipatterns

### Inventorial CLAUDE.md

The failure mode: file layouts, tech-stack lists, or "current state" copy accumulate in this file and go stale within weeks.

**Do instead:** structural content goes into prime reads (live each session) or `docs/` (versioned). CLAUDE.md holds only durable role, rules, and antipatterns.

**Reason:** single-layer persistence designs fail. The three-way contract (prime / CLAUDE.md / close) exists so each layer owns one kind of artifact; blurring it collapses the design.

### Cross-harness drift

The failure mode: a Claude Code primitive is added to `.claude/` and the Pi equivalent is never built, so the project silently locks to one harness.

**Do instead:** when a CC primitive ships, open a follow-up task for the Pi mirror and surface it at the next `/rag-web-close`.

**Reason:** README requirement #1 commits the project to supporting both harnesses. Silent single-harness lock-in violates that commitment.

### Auto-commit on close

The failure mode: `/rag-web-close` or a dispatched agent commits changes without operator review.

**Do instead:** present uncommitted changes with options (commit as-is, commit selected paths, discard specific changes, leave for next session). Wait for the operator.

**Reason:** close is read + orchestrate, never write-by-itself. The commit gate belongs to the operator.
